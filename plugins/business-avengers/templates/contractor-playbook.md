## When to Hire Contractors

### Decision Framework

**Hire a Contractor When:**
- Task is repetitive but too complex/expensive to automate
- Task requires human judgment but not your specific expertise
- Task is time-consuming but low strategic value
- Task occurs predictably but not frequently enough for full-time hire
- Task can be clearly defined with success criteria

**Don't Hire (Automate Instead) When:**
- Task is 100% rule-based with no edge cases
- Task can be automated in less time than it takes to hire/train
- Task frequency is high (daily/hourly) and volume is predictable

**Don't Hire (Do It Yourself) When:**
- Task is strategic and requires deep business context
- Task involves sensitive data or critical decisions
- Task is faster for you to do than to explain
- Task builds skills you need long-term

## Offboarding Process

### Planned Offboarding (End of Contract)

**2 Weeks Before:**
- [ ] Notify contractor of non-renewal
- [ ] Document all in-progress work
- [ ] Request final deliverables list

**1 Week Before:**
- [ ] Knowledge transfer session
- [ ] Update SOPs with any tribal knowledge
- [ ] Request return of any physical assets

**Last Day:**
- [ ] Final invoice/payment
- [ ] Revoke all access
- [ ] Delete credentials from systems
- [ ] Archive communication history
- [ ] Request LinkedIn recommendation (if applicable)

### Emergency Offboarding (Performance Issues)

**Immediate:**
- [ ] Revoke access to critical systems
- [ ] Change shared passwords
- [ ] Document reason for termination
- [ ] Review work for quality/security issues
- [ ] Process final payment per contract terms
